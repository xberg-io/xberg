# Upstream Sync Strategy

Upstream is `xberg-io/xberg`, reachable via git remote `upstream`. `origin` is
`jamon8888/xberg` — a **fork**, not upstream; `origin/main` already carries all of this
fork's RAG/gliner/doc-store additions and is not a mirror of anything upstream.
Never edit `CLAUDE.md` directly — it is generated by `ai-rulez generate` from `.ai-rulez/`.

## Fence Model

`crates/xberg-rag` (the RAG base crate itself) is already part of upstream
`xberg-io/xberg` — it is not a fork-only addition. Do not describe RAG-the-crate as ours.

Fork-only crates (verified absent from `xberg-io/xberg`):

- `xberg-rag-node`, `xberg-doc-store`, `xberg-gliner-candle`, `xberg-pdfium-render`

Fork-only packages/dirs:

- `packages/xberg-wasm-runtime`, `packages/xberg-web-ui`, `mcp-server/`, `docs/superpowers/`

The fork's changes inside upstream-owned `crates/xberg-rag` (graphqlite backend + the
`Graph` `RetrieveMode` variant) are a tracked **carry-patch**, not a clean addition — see
`docs/superpowers/upstream/carry-patches.tsv`.

The `/v1/process` + rehydrate API surface is fenced behind the `process-api` cargo feature
in `crates/xberg/src/api/rag/` — this is the model for isolating fork API additions inside
upstream-owned code.

## Cargo.toml Fence Rule

Fork-only workspace members, `[workspace.dependencies]` entries, and `[patch.crates-io]`
entries live in an append-only block at the end of each list, after a
`# === fork overlay ===` comment. Upstream entries are never reordered.

## Sync Mechanics

See `docs/superpowers/upstream/sync.md` (selective-sync runbook) and
`docs/superpowers/upstream/carry-patches.md` (Tier-2 tracked files) for the full procedure
to pull upstream changes and reapply carry-patches.

## Regenerate CLAUDE.md After Editing Rules

```sh
# Edit files in .ai-rulez/ then:
ai-rulez generate
# or via task:
task ai-rulez:generate

# Commit both source and generated file together:
git add .ai-rulez/ CLAUDE.md
git commit -m "docs(ai-rulez): ..."
```

## What NOT to Change in Upstream Files

- `crates/xberg/` — never modify; pull changes from upstream directly
- `packages/*/` — Alef-generated; run `task alef:generate` instead of hand-editing
- `CLAUDE.md` — generated; edit `.ai-rulez/` source files instead
