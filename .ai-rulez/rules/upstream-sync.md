# Upstream Sync Strategy

This repo is a fork of `jamon8888/xberg` (origin remote). We carry local additions on top.
Never edit `CLAUDE.md` directly — it is generated by `ai-rulez generate` from `.ai-rulez/`.

## Branch Model

Work happens on `feature/rag-mcp` (or equivalent feature branch). `main` tracks upstream.

```
main          ← mirrors origin/main (jamon8888/xberg)
feature/rag-mcp  ← our work, rebased on top of main
```

## Our Additions (owned by us, not upstream)

| Path | Description | Conflict risk |
|------|-------------|---------------|
| `crates/xberg-rag/` | RAG base layer (we added `b9dc796bf2`) | Low — upstream doesn't own it |
| `crates/xberg-rag-node/` | NAPI-RS bindings for RAG | Zero — new crate |
| `mcp-server/` | MCP TypeScript server (30 tools) | Zero — new directory |
| `docs/superpowers/` | Implementation plans and specs | Zero — new directory |

## Files With Conflict Surface

| File | What we changed | Action on conflict |
|------|----------------|-------------------|
| `Cargo.toml` | Added `xberg-rag-node` workspace member | Manual merge (append-only) |
| `crates/xberg-rag/src/query.rs` | Added `Graph` variant to `RetrieveMode` | Reapply our enum variant |
| `crates/xberg-rag/src/backends/sqlite.rs` | Wired graphqlite backend | Reapply graphqlite dispatch block |
| `.cargo/config.toml` | Windows build paths | Keep ours, verify upstream didn't change differently |

`crates/xberg/` (core extraction engine) — **zero changes**, always conflict-free.

## Pull Upstream Changes

```sh
git fetch origin
git checkout feature/rag-mcp
git rebase origin/main
# Resolve any conflicts in the files listed above, then:
git rebase --continue
```

For `Cargo.toml` conflicts: keep both workspace member lists, remove conflict markers.

## Regenerate CLAUDE.md After Editing Rules

```sh
# Edit files in .ai-rulez/ then:
ai-rulez generate
# or via task:
task ai-rulez:generate

# Commit both source and generated file together:
git add .ai-rulez/ CLAUDE.md
git commit -m "docs(ai-rulez): ..."
```

## What NOT to Change in Upstream Files

- `crates/xberg/` — never modify; pull changes from upstream directly
- `packages/*/` — Alef-generated; run `task alef:generate` instead of hand-editing
- `CLAUDE.md` — generated; edit `.ai-rulez/` source files instead
