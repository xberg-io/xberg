## Test Generation

Most test apps have their e2e test files auto-generated by the `kreuzberg-e2e-generator` tool from fixture definitions in `fixtures/`.

```bash
# Regenerate tests for a specific language
cargo run -p kreuzberg-e2e-generator -- generate --lang <lang> --fixtures fixtures --output tests/test_apps

# Supported languages: java, go, rust, python, typescript, ruby, php, csharp, elixir, wasm_deno, wasm_workers
```

**Important:** The generator writes test files into `tests/test_apps/<lang>/`. Some test apps (e.g. Java, Go) have hand-written files (pom.xml, go.mod, main app code) that should NOT be overwritten. After regeneration, verify with `git diff` that only test files changed.

## Test Documents

Tests resolve documents from the main `test_documents/` directory at the kreuzberg repo root. Key subdirectories:

| Directory | Contents |
|-----------|----------|
| `pdf/` | PDF files (tiny, medium, large, OCR test PDFs) |
| `docx/` | Word documents |
| `xlsx/` | Excel spreadsheets |
| `images/` | JPEG, PNG, BMP, SVG, JP2 images |
| `odt/` | OpenDocument Text |
| `markdown/` | Markdown files |
| `html/` | HTML files |
| `epub/` | EPUB books |
| `email/` | EML files |
| `archives/` | ZIP, TAR, GZ archives |
