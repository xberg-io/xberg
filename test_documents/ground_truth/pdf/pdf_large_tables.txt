Abstract

Recently, significant progress has been made applying deep learning to the problem of table detection and extraction from unstructured documents. However, one significant hindrance to further progress is the scarcity of datasets and baselines for training table extraction algorithms. In this work, we develop a new comprehensive dataset that includes detailed annotations including table location, structure, and functional analysis of tables. We produce a large-scale open source PubTables-1M dataset that contains detailed annotations for over one million tables extracted from scientific papers in the PubMed Central Open Access (PMCOA) repository, which is freely available. We also present detailed baselines that address a significant array of ground truth consistency issues that may arise when creating large-scale table datasets. This work is significant because: (1) PubTables-1M provides a large-scale dataset for table detection and extraction from documents, (2) table structure recognition (TSR) and table functional analysis (FA) is challenging for automated systems with ~50% PubTables-1M's dataset of tables with simple structure being recognized by humans. Our method results in detailed annotations of tables to make it easily understandable in their extraction and evaluation process. 9.1. Model Details

For both DETR models, we use a ResNet-18 backbone, 200 layers in the encoder, and six layers in the decoder. For the experimental setup in all experiments, we use a learning rate of 1e-4, 0.125 epochs, each chosen to be slightly over the
maximum number of objects in each set's training samples. Besides this, we use the same configuration in both DETR and DETR-SC setups.

All of the experiments are performed using a single NVIDIA Tesla V100 GPU. The training on the dataset with 30 epochs using all default hyperparameters and standard 300-dimensional model weight decay and momentum optimization. We used the same parameters for FA training on 3000 images, as well as initial learning rate of 0.0005.

We use the default hyperparameters of fine-tuning configuration parameters for the transfer learning from initial trained states of HARC, and HARC and where we fine-tune each model that had the loss than in the previous set after one epoch of training.

9.2. Inference

While the model is trained mostly as an object detector, in inference time we applied box-level inference and pruning to objects supported by the model in a structured table. Objects in our model from hierarchical relationships, shape-based. Several sets for instance we focus on the model's predictions for physical constraints, or overlap. For details on the full set of relationships in the hierarchy, please see the code. One key observation is that when the structural information between cells, one may parent objects of the same state. Therefore, a conflict occurs when two parent objects of the same state are determined to be not completely connected with a parent object's bounding box.

For example, two spanning cell objects are considered to conflict if they both overlap with the same grid cell. The bounding box-based pruning step resolves the conflict by selecting the object with the highest intersection with area by overlapping box weights.

This creates the canonical data, and the corrected data is used for training time the model learns to predict output that is also free of conflicts. However, to address any conflicts produced by the model, we add to the model's a simple conflict resolution step. The

conflict resolution step resolves overlapping objects or adjusts the hierarchical inter-relationships such that when an object is determined to be in conflict with another object, we suppress, except spacing between objects is defined in terms of shared edge pixels. Hence the position and size of objects are computed from their child objects.
