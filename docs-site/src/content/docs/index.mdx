## Why Xberg

<CardGrid>
	<Card title="One engine for everything" icon="random">
		Feed documents, URLs, code, images, and audio into one API and get clean Markdown, entities, code
		structure, and embeddings back — no stitching libraries together.
	</Card>
	<Card title="Extract from any document type" icon="document">
		PDFs, Office files, images, email, archives, and academic papers all come out as clean Markdown or
		one of five other formats, with no per-format setup.
	</Card>
	<Card title="Text from images and audio, automatically" icon="magnifier">
		Scanned pages and spoken audio become searchable text with confidence scores and language
		detection, falling back across OCR and transcription backends when one struggles.
	</Card>
	<Card title="Get structured data without custom prompting" icon="laptop">
		Pull entities and JSON that matches your schema straight from any document, using a local or
		hosted LLM — no prompt engineering.
	</Card>
	<Card title="Understand code across 306 languages" icon="setting">
		Extract functions, classes, imports, and symbols from source, then turn any content into
		embeddings for search and RAG.
	</Card>
	<Card title="Use your language, native performance" icon="translate">
		Call Xberg from Python, TypeScript, Rust, Go, Java, C#, Ruby, PHP, Elixir, and more — plus a CLI,
		REST API, MCP server, Docker, and WASM.
	</Card>
</CardGrid>

:::tip[Choosing between TypeScript packages]
**`@xberg-io/xberg`** — Native NAPI-RS bindings. Use for Node.js servers and CLI tools. Full
feature set at native performance.

**`@xberg-io/xberg-wasm`** — Pure WebAssembly. Use for browsers, Cloudflare Workers, Deno, Bun, and
serverless environments (cross-platform, 60–80% of native speed).
:::

## Quick example

<Tabs syncKey="lang">
<TabItem label="Python">

```python title="main.py"
from xberg import ExtractInput, extract

output = await extract(ExtractInput(kind="uri", uri="document.pdf"))
print(output.results[0].content)
```

</TabItem>
<TabItem label="TypeScript">

```typescript title="index.ts"
import { ExtractInputKind, extract } from "@xberg-io/xberg";

const output = await extract({
  kind: ExtractInputKind.Uri,
  uri: "document.pdf",
});
console.log(output.results[0].content);
```

</TabItem>
<TabItem label="Rust">

```rust title="src/main.rs"
use xberg::{extract, ExtractInput, ExtractionConfig};

let config = ExtractionConfig::default();
let output = extract(ExtractInput::from_uri("document.pdf"), &config).await?;
println!("{}", output.results[0].content);
```

</TabItem>
</Tabs>
